# Editorial Rules

Voice:

- Ukrainian by default.
- Direct, practical, founder/operator tone.
- No generic AI hype.
- No fake certainty, no invented metrics.
- Prefer concrete examples, workflows, prompts, tradeoffs.
- Keep posts readable as standalone Threads posts.
- For news, default to a real thread/chain, not a compressed single post.

Strong formats:

- `news thread`: hook -> context -> why it matters -> implications -> practical takeaway
- `myth`: common belief -> correction -> useful rule
- `case`: situation -> bottleneck -> AI move -> result shape
- `howto`: problem -> steps -> CTA
- `opinion`: clear take -> reason -> implication
- `checklist`: quick diagnostic list
- `comparison`: option A vs B vs C -> when to use

Avoid:

- "AI will change everything"
- "You need to understand one thing"
- Overusing emoji
- Hashtag stuffing
- More than one CTA
- Compressing a news item with real context into one post unless explicitly asked
